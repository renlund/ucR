Underlying Idea

Data (numerical, factors, text, statistics thereof) is mapped onto aestethics (horizontal-, vertical axis, color, shape, size, etc.).

The mapping in conjuction with a geometry and scales determine the graph.

Basics

Get some data

DF <- subset(mtcars, select = c(1:4, 8))
DF$cyl <- factor(DF$cyl, labels = LETTERS[1:3])
DF$vs <- factor(DF$vs, labels = c("Noll", "Ett"))
str(DF)

Create an Aestethic Mapping

p <- ggplot(data = DF, mapping = aes(x = mpg, y = hp))
p

## Error: No layers in plot

# str(p)

There is no plot yet! We have to add a geometry. 

p + geom_point()

The point geometry comes with a number of defaults. E.g. there is a default shape, size, edge-color and fill-color. The x- and y-axis are continuous in a cartesian coordinate system.

p + geom_point(shape = 21, fill = "black", color = "black", alpha = 1) + coord_cartesian() + 
    scale_x_continuous() + scale_y_continuous()

We can switch to polar cordinates (but it doesn't make much sense here):

p + geom_point() + coord_polar(theta = "x")

There is a neat built-in way to visualize overlapping points; change the transparency of the points with 'alpha'.

p + geom_point(size = 5, alpha = 0.25)

There are also options for changing the scales of the axis;

p + geom_point() + scale_x_log10()

Here the defaults are not sufficient to make the x-axis clear;

p + geom_point() + scale_x_log10(limit = c(10, 100), breaks = c(10, 20, 40, 
    80))

Tweaking is available.

p + geom_point() + labs(x = "The Majestic X", y = "The Amazing Y", title = "Oh, ggplot2, is there anything you can't do?") + 
    theme_bw()

More aestethics

Now, lets map variable 'cyl' to color and 'disp' to size. 

p <- ggplot(DF, aes(mpg, hp, color = cyl, size = disp)) + theme_bw() + labs(x = expression(frac(Miles, 
    Gallon)), y = "Horsepower")
p + geom_point()

Easy control over each scale;

p + geom_point(alpha=0.5) + 
   scale_size(
      range=c(4,9),   # actual size in plot
      # limits = c(200,300), 
      breaks=c(100,250,400), 
      # labels=c("Small", "Medium", "Huuuge!"),
      name="Displacement"
      ) + 
   scale_color_hue( # mapping to the HCL space 
#       h = c(0,360), # hue
#       c = 60,       # chroma
#       l = 60,       # luminance 
#       h.start = 0,  
#       direction = 1, 
      limits = c("A", "B", "C", "Whatever"),
      labels = c("Type A", "Type B", "Type C", "Missing"), 
      name = "Cylinder \ntype"
      )

There are other scales available for color and size.

display.brewer.all(n = 3)

E.g. color can be set according to some 'Color Brewer' palette and size can be scaled according to area. 

p + geom_point(alpha = 0.6) + scale_size_area(max_size = 9) + scale_color_brewer(palette = "Dark2")

The same variable can be mapped to more than one aestetics.

p <- ggplot(DF, aes(mpg, hp, col = mpg, size = disp, shape = vs)) + geom_point() + 
    theme_bw() + scale_size_area(max_size = 8)
p

Continuous color scale can be manipulated. 'scale_color_gradient' sets up a sequential scale.

p + scale_color_gradient(low = "green", high = "red")

'scale_color_gradient2' sets up a divergent scale.

p + scale_color_gradient2(low = "green", mid = "blue", high = "red", midpoint = 22)

Mapping constants

If you just want a color, say black, for you graph, why not map color to “black”? This will essentially create a new variable with the value 'black' for individuals in the data frame and this variable will be mapped to color (but not by default the color black).

ggplot(data = DF, mapping = aes(x = mpg, y = hp, color = "black")) + geom_point()

However, if you do have a variable with color names, these can be used with the identity scale. Also, if you want to add some detail to your plot you could map constants to the x- and y axis. 

ggplot(data = DF, mapping = aes(x = mpg, y = hp, color = "black")) + geom_point() + 
    scale_color_identity() + geom_text(aes(x = 30, y = 200), label = "HEJ!", 
    size = 15) + geom_point(aes(x = 30, y = 250), col = "green", size = 10)

But; this (as far as I understand it) accomplishes this feat by creating new constant variables (which is somewhat wasteful). Better to use 'annotate' 

ggplot(data = DF, mapping = aes(x = mpg, y = hp, color = "black")) + geom_point() + 
    scale_color_identity() + annotate("text", x = 30, y = 200, label = "HEY!", 
    size = 17) + annotate("segment", x = 20, xend = 30, y = 250, yend = 300, 
    col = "red") + annotate("rect", xmin = 15, xmax = 20, ymin = 150, ymax = 350, 
    col = "blue", fill = "lightblue", alpha = 0.3)

Adding statistics

Mapping 'cyl' to color creates subgroups that is inherited by all subsequent layers.

p <- ggplot(DF, aes(mpg, hp, color = cyl)) + theme_bw()
p + geom_point(size = 4) + stat_smooth(method = "lm")

To have an abline for all points, you can set to color mapping for 'points' only.

p <- ggplot(DF, aes(mpg, hp)) + theme_bw()
p + geom_point(aes(color = cyl), size = 4) + stat_smooth(method = "lm")

Or you can override the previously set aestethic mapping

p <- ggplot(DF, aes(mpg, hp, col = cyl)) + theme_bw()
p + geom_point(size = 4) + stat_smooth(aes(x = mpg, y = hp), method = "lm", 
    formula = y ~ poly(x, 2), color = "black", lty = 3, fill = "purple", alpha = 0.2)

Danger! Scales cannot be used to zoom!

p <- ggplot(DF, aes(mpg, hp)) + theme_bw() + geom_point(aes(color = cyl), size = 4) + 
    stat_smooth(aes(x = mpg, y = hp), method = "lm")
p + scale_x_continuous(lim = c(17, 25))

## Warning: Removed 17 rows containing missing values (stat_smooth).
## Warning: Removed 17 rows containing missing values (geom_point).

For that you need 'coord_cartesian':

p + coord_cartesian(xlim = c(17, 25))

A problem with ggplot2 is that you necessarily need your variables (for an aestethic mapping) in a single dataframe BUT you can add layers with data from another.

p + geom_line(data = data.frame(q = seq(10, 35, len = 20), noise = rnorm(20, 
    200, 150)), aes(x = q, y = noise))

Other

Dotplot-ish

Another way to deal with 'overplotting'

p <- ggplot(DF, aes(x = cyl, y = mpg, col = cyl))
p + geom_point()

is by adding random noise (plus, remove the superfluous color scale legend):

p + geom_point(position = "jitter", size = 3) + scale_x_discrete(limits = c("A", 
    "C", "B"), lab = c("Typ A", "Typ C", "Typ B")) + scale_color_hue(guide = FALSE)

I thought the above was a dotplot, but ggplot has something else in mind:

p + geom_dotplot()  # y-axis?

which, in its default mode, ignores the mapping of 'mpg' to the y-axis. It can however, take the 'mpg' mapping into account and stack it's point in another direction.

p <- ggplot(DF, aes(x=cyl, y=mpg, fill=cyl))
p + geom_dotplot(
   # binwidth=1,  # dotplot is essentially a histogram
   col="black", 
   binaxis="y",
   stackdir="up"
   )

## stat_bindot: binwidth defaulted to range/30. Use 'binwidth = x' to adjust this.

One can make violin plots

p + geom_violin()

and add a dotplot layer:

p + geom_violin() + geom_dotplot(binaxis = "y", stackdir = "center", fill = "white")

## stat_bindot: binwidth defaulted to range/30. Use 'binwidth = x' to adjust this.

One can have a boxplot statistical layer.

p <- ggplot(DF, aes(x = cyl, y = mpg, fill = cyl)) + scale_fill_hue(guide = FALSE)
p + stat_boxplot(outlier.shape = 3)

The line between geometrical- and statistical layers are fuzzy:

p + geom_boxplot(outlier.shape = 3)

All geoms and stats are convenience functions setting up a more complicated function 'layer' (which noone seems to use)

ggplot(data = DF, mapping = aes(x = cyl, y = mpg)) + layer(geom = "boxplot", 
    geom_params = list(fill = "lightblue"), stat = "boxplot", stat_param = list())

Each convience 'geom' has a default 'stat' and vice versa.

In stead of making things more complicated with 'layer' there is an option to 'quick-plot', which (often) provides a geometry if none is given

qplot(x = cyl, y = mpg, col = cyl, data = DF)

or take 'geom' as a argument. 

qplot(x = cyl, y = mpg, fill = cyl, data = DF, geom = "boxplot")

More on statistical layers

You can apply functions to create statistics - as long as they return named values corresponding to the choosen geom.

p <- ggplot(data=DF, mapping=aes(x=cyl, y=mpg, col=cyl)) + theme_bw()
p + geom_point(position="jitter", width=0.5) +
   stat_summary(
      fun.data=mean_cl_normal, #wrapper for smean.cl.normal (Hmisc)
      geom="errorbar",
      width=0.3,
      lwd=2
      )

E.g. for geom 'errorbar' and 'crossbar' you need a function that returns 'y', 'ymin' and 'ymax'. 

myF <- function(x) {
    W <- wilcox.test(x, conf.int = TRUE, exact = FALSE)
    r <- c(W$estimate, W$conf.int[1], W$conf.int[2])
    names(r) <- c("y", "ymin", "ymax")
    return(r)
}
p + geom_point(position = "jitter", width = 0.5) + stat_summary(fun.data = myF, 
    geom = "crossbar", width = 0.3, )

Histograms and Density function estimates

A histogram can be produced via

p <- ggplot(DF, aes(x = mpg))
p + geom_histogram()  # Notice the warning; the default number of bins is consciously set inadequatly

## stat_bin: binwidth defaulted to range/30. Use 'binwidth = x' to adjust this.

But to see some more features of ggplot2:

p <- ggplot(DF, aes(x = mpg))
p + geom_histogram(aes(y = ..count..), binwidth = 2)

The '..some-statistic..' can be used to extract statistics. (I assume these statistics are created by the associated 'stat_bin' - documentation is scarce!).

require(scales)  # for 'labels=percent')
# help(package='scales')
p + geom_histogram(aes(y = ..count../sum(..count..)), binwidth = 2) + scale_y_continuous(labels = percent)

How to get a 'density' histogram with estimated and theoretical distribution:

p + geom_histogram(aes(y = ..density..), binwidth = 2, fill = "lightblue") + 
    geom_density(col = "blue") + stat_function(fun = dnorm, args = c(mean = 18, 
    sd = 4.5), col = "red", lty = 2)

Histograms and kernel estimates can be overlapping

p <- ggplot(DF, aes(x = mpg, fill = cyl))
# p + geom_bar(binwidth=2, col='black', position='identity', alpha=1)
p + geom_bar(binwidth = 2, col = "black", position = "identity", alpha = 0.5)


# p + geom_density(position='identity', alpha=1)
p + geom_density(position = "identity", alpha = 0.5)

or stacked.

p + geom_bar(binwidth = 2, col = "black", position = "stack")


p + geom_density(position = "stack")

Examples

Build a ROC curve

# use dataset 'esoph' from the 'datasets' package
mod <- glm(cbind(ncases, ncontrols)~agegp+alcgp+tobgp, data=esoph, family=binomial) # fit a logistic regression model
fit <- fitted(mod) # save the estimated probabilities
N <- dim(esoph)[1]
sens <- rep(NA_real_, N) # container for sensitivy calculation
spec <- rep(NA_real_, N) # container for specificity calculation
attach(esoph)
for(k in 1:N){
   filter <- fit >= fit[k] 
   sens[k] <- sum(ncases[filter])/sum(ncases)
   spec[k] <- sum(ncontrols[!filter])/sum(ncontrols)
   }
detach(esoph)
DATA <- as.data.frame(cbind(sens, spec, fit=fitted(mod))) 

sens <- sens[order(spec)]  # for calculating the AUC
spec <- spec[order(spec)]
auc <- ( sum(sens[1:(N-1)]*diff(spec)) + sum(sens[2:N]*diff(spec)) ) / 2 # estimate the area under the ROC curve
p <- ggplot(DATA, aes(x=spec, y=sens, col=fit))
p + geom_line(size=2) + 
   scale_x_reverse() + # flip the x-axis
   scale_color_gradient(low="pink", high="black", name="Cut Off") + 
   labs(
      x="Specificity",
      y="Sensitivity",
      title="ROC curve"
      ) +
   geom_abline(intercept=1, slope=1, lty=2) + # ?
   annotate(geom="text", label=paste("AUC:", round(auc,2)), x=0.3, y=0.25)

A survival curve

require(survival)
KIDN <- read.table(file = "http://lib.stat.cmu.edu/datasets/kidney", header = FALSE, 
    sep = "", skip = 17)
names(KIDN) <- c("id", "time", "status", "age", "sex", "dis_type", "est_frail")
KIDN$sex <- factor(KIDN$sex, labels = c("Male", "Female"))
# str(KIDN)
mod <- coxph(Surv(time = time, event = status) ~ strata(sex), data = KIDN)
x <- survfit(mod)
# str(x)
with(x, DATA <<- data.frame(x = time, y = surv, gender = factor(c(rep("Male", 
    strata[1]), rep("Female", strata[2]))), at_risk = n.risk))
p <- ggplot(DATA, aes(x, y, color = gender)) + labs(x = "Time", y = "Survival")
p + geom_point(mapping = aes(x, y, size = at_risk)) + geom_line(lty = 2, alpha = 0.5) + 
    scale_color_hue(name = "Gender") + scale_size_area(max_size = 5, name = "At risk", 
    breaks = c(1, 12, 23, 34, 45, 56)) + scale_y_continuous(labels = percent) + 
    theme(legend.position = "bottom", legend.direction = "horizontal")
